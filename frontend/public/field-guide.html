SIPJOLT Field Validation Guide

v1.01 • Machine API Check
📱 PHASE 1: APP LOGIN & CONNECTIVITY 2 min
1App Launch & Login
Open SIPJOLT App → Enter 4-digit Code → Ops Command Center
  • Open the SIPJOLT app on your device
  • Enter Ops Manager code: 4782
  • Wait for dashboard to load
Dashboard loads with dark theme, sidebar visible on left
2Diagnostics Health Check
Sidebar → Diagnostics
  • Click Diagnostics in the sidebar (bottom)
  • Check status indicators for: Yile API, Database
  • Run the test buttons if available
All indicators show GREEN / "healthy" status
If Yile API shows RED, machine commands will not work!
🗺️ PHASE 2: FLEET DASHBOARD & DEVICE 3 min
3Fleet Map Loading
Sidebar → Fleet
  • Click Fleet in the sidebar
  • Wait for map to load (may take a few seconds)
  • Look for machine markers on the map
Map displays with at least 1 device marker visible
4Device Details Modal
  • Tap/click the device marker for your machine
  • Click "View Details" in the popup
  • Verify all 4 tabs are present: Overview, Inventory, Recipes, Sales
Modal opens with 4 tabs, device status shows "Normal" or "Online"
5Inventory Tab Data
  • Click the Inventory tab
  • Check that ingredient levels display with percentages
  • Compare app levels to physical machine display
Ingredient bars show % levels (e.g., Coffee Beans 75%)
Levels should roughly match what's shown on the machine screen
📦 PHASE 3: REPLENISHMENT CENTER 2 min
6Replenishment Sidebar Access
Sidebar → Replenishment
  • Click Replenishment in the sidebar
  • Select your device from the dropdown
  • Wait for inventory data to load
Inventory Levels panel displays with all ingredients listed
7Ingredient Mapping Check
  • Verify these ingredients appear (if machine has them):
  • Coffee Beans, Oat Milk, Matcha, Cocoa, Chai
  • Vanilla Syrup, Caramel Syrup, Hazelnut Syrup, Brown Sugar Syrup
  • Cups, Hot Water
Ingredients show with correct names (not Chinese characters)
🎮 PHASE 4: MACHINE CONTROL 5 min
8Machine Control Panel
Sidebar → Machine Control
  • Click Machine Control in the sidebar
  • Verify device dropdown shows your machine
  • Verify recipe dropdown populates with drinks
  • Check status badge shows ONLINE (green) or OFFLINE (red)
Both dropdowns have options, status shows ONLINE
9Test Brew Command
  • Select a simple recipe (e.g., Americano or Espresso)
  • Click the green Brew button
  • Watch the machine - it should start brewing within 10 seconds
  • Check app shows success message: "☕ Brew started!"
Machine begins brewing, app shows success message
Have a cup ready! The machine will dispense the drink.
10Take Offline Toggle
  • Click the red Take Offline button (WiFi-off icon)
  • Status badge should change to "OFFLINE" (red)
  • Brew button should become disabled (grayed out)
  • Click the button again to bring machine back online
Status toggles between ONLINE/OFFLINE, Brew button enables/disables
This is for maintenance mode - use when servicing the machine
✅ PHASE 5: FINAL VERIFICATION 1 min
11Physical Machine Check
  • Check machine screen shows "Ready" or similar idle state
  • Verify machine WiFi indicator shows connected
  • Confirm no error messages on machine display
Machine is idle, connected, and error-free

🔧 Troubleshooting Guide (Click to Expand)
❌ Diagnostics shows RED for Yile API
  1. Check machine is powered on and connected to WiFi
  2. Wait 2 minutes and refresh the page
  3. If still red, contact Michael immediately
❌ No devices appear on Fleet map
  1. Refresh the page (pull down on mobile)
  2. Check your internet connection
  3. Try logging out and back in
❌ Recipe dropdown is empty
  1. Select a different device, then select your device again
  2. Wait 5 seconds for recipes to load
  3. If still empty, the recipe catalog may need syncing
❌ Brew command sent but machine doesn't respond
  1. Check machine isn't in cleaning or maintenance mode
  2. Verify machine screen shows "Ready" state
  3. Power cycle machine: OFF for 30 sec, then ON
  4. Wait 2 minutes for reconnection, then retry
❌ Inventory shows 0% for all ingredients
  1. This usually means API sync is delayed
  2. Click the refresh button in Replenishment
  3. Wait 30 seconds and check again
❌ Status shows "UNKNOWN"
  1. Machine may have just come online
  2. Refresh the page to get latest status
  3. Check physical machine shows connected
